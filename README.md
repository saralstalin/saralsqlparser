# SaralSQL Parser

SaralSQL is a high-fidelity, recursive-descent T-SQL parser built in TypeScript. Unlike standard SQL parsers designed for simple query execution, SaralSQL is architected specifically to power **Language Server Protocol (LSP)** features such as IntelliSense, precise error recovery, symbol renaming, and code navigation.

> [!IMPORTANT]  
> **Status:** This package is currently in active development and is **not yet published to npm**.

## 🎯 The Goal
The primary objective of SaralSQL is to provide the structural and physical context required by modern IDEs (like VS Code and Azure Data Studio). 

While many parsers discard "trivial" data like whitespace or exact offsets, SaralSQL preserves the relationship between the code and the document. This ensures that every AST node corresponds perfectly to the user's cursor position, enabling reliable editor tooling.

---

## ✨ Features (What is Done)

### Precision Tracking & Compatibility
* **Byte-Perfect NodeLocation:** Every node (Statements, Expressions, Join clauses, Columns) carries exact `start` and `end` byte offsets.
* **Windows-Aware Lexing:** Dedicated handling for `\r\n` carriage returns to prevent "position drift" in scripts generated by SSMS.
* **Multipart Identifier Resolution:** Sophisticated parsing for T-SQL naming conventions, including `[schema].[table]`, `#TempTables`, and `@Variables`.

### Core DML Support
* **SELECT:** Full support for `TOP`, `DISTINCT`, complex `JOIN` sequences, `GROUP BY`, `HAVING`, and `ORDER BY`.
* **INSERT:** Handles `INSERT INTO...VALUES` and `INSERT INTO...SELECT` patterns.
* **UPDATE:** Supports T-SQL’s unique `UPDATE...FROM...JOIN` syntax.
* **DELETE:** Supports both standard deletes and alias-based join deletes (`DELETE u FROM Users u...`).

### Advanced Expression Engine
* **Refined Pratt Parser:** A high-precedence expression engine handling complex math, boolean logic, and T-SQL specifics.
* **Normalized Tokenization:** Lexer-level normalization (Keywords to UpperCase) for high-performance parsing and consistent comparison.
* **Specialized Operators:** Full support for `BETWEEN`, `IN`, `LIKE`, `IS NULL`, `EXISTS`, and `CASE` expressions.
* **Composite Operators:** Correctly merges multi-token operators like `>=`, `<=`, `<>`, and `!=` at the Lexer level.
* **Window Functions:** Support for `OVER (PARTITION BY ... ORDER BY ...)` clauses.

---

## 🛠 Project Structure
* **Lexer:** Tokenizes raw T-SQL into a stream of typed tokens while maintaining line and column metadata.
* **Parser:** A recursive-descent parser that transforms tokens into a strongly-typed Abstract Syntax Tree (AST).
* **Types:** Rigorous TypeScript interfaces ensuring every node adheres to a strict location contract.

## ⏳ What is Pending (The Roadmap)

### Phase 1: Structural Intelligence (Immediate Focus)
* [ ] **Symbol Table & Scoping:** Implement a `ScopeStack` to track Common Table Expressions (CTEs) and local variables (`DECLARE`).
* [ ] **CTE Resolution:** Allow the parser to "link" a table reference in a query to a definition in a `WITH` clause.
* [ ] **Type Inference:** Basic engine to determine the result type of expressions for IntelliSense tooltips.

### Phase 2: Advanced T-SQL Idioms
* [ ] **The OUTPUT Clause:** Support for `INSERT/UPDATE/DELETE ... OUTPUT` which is critical for complex ETL.
* [ ] **MERGE Statement:** Implementation of the high-complexity `MERGE` syntax.
* [ ] **Batch Processing:** Support for `GO` batch separators and cross-batch variable lifecycle.

### Phase 3: DDL & Maintenance
* [ ] **Schema Objects:** Expanding support for `CREATE/ALTER/DROP` for Tables, Views, and Stored Procedures.
* [ ] **Full Script Validation:** Building a "Linter" mode to identify anti-patterns in T-SQL.
