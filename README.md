### SaralSQL Parser

SaralSQL is a high-fidelity, recursive-descent T-SQL parser built in TypeScript. Unlike standard SQL parsers designed for simple query execution, SaralSQL is architected specifically to power **Language Server Protocol (LSP)** features such as IntelliSense, precise error recovery, symbol renaming, and code navigation.

> [!IMPORTANT]
> **Status:** This package is currently in active development and powers the SaralSQL VS Code extension. It is not yet published as a standalone npm package.

---

## 🎯 The Goal

The primary objective of SaralSQL is to provide the structural and physical context required by modern IDEs (like VS Code and Azure Data Studio). 

While many parsers discard "trivial" data like whitespace or exact offsets, SaralSQL preserves the relationship between the code and the document. This ensures that every AST node corresponds perfectly to the user's cursor position, enabling reliable editor tooling.

---

## ✨ Features (Current Progress)

### Precision Tracking & Fault Tolerance
* **Byte-Perfect NodeLocation:** Every node (Statements, Expressions, Join clauses, Columns) carries exact `start` and `end` byte offsets.
* **Fault-Tolerant Parsing:** Includes a `resync()` mechanism that allows the parser to recover from syntax errors (like a missing `FROM` clause or "garbage" tokens) and continue parsing the rest of the script.
* **Windows-Aware Lexing:** Dedicated handling for `\r\n` carriage returns to prevent "position drift" in scripts generated by SSMS.

### Core T-SQL Support
* **DML (Data Manipulation):** Full support for `SELECT` (including `TOP`, `DISTINCT`, `JOIN` sequences), `INSERT`, `UPDATE`, and `DELETE` (including alias-based join deletes).
* **DDL (Data Definition):** Robust support for `CREATE PROCEDURE`, `FUNCTION`, `VIEW`, and `TABLE`.
* **Control Flow & Blocks:** Comprehensive parsing of `IF...ELSE` logic and `BEGIN...END` blocks, essential for script-based T-SQL logic.
* **Batch Support:** Recognizes `GO` separators to isolate errors and handle multi-batch script lifecycles.

### Advanced Expression Engine
* **Refined Pratt Parser:** A high-precedence engine handling math, boolean logic, and T-SQL specifics like `BETWEEN`, `IN`, `LIKE`, and `CASE`.
* **Window Functions:** Full support for `OVER (PARTITION BY ... ORDER BY ...)` clauses.
* **Multipart Identifier Resolution:** Sophisticated parsing for `[schema].[table]`, `#TempTables`, and `@Variables`.

---

## 🛠 Project Structure

* **Lexer:** Tokenizes raw T-SQL into a stream of typed tokens while maintaining line and column metadata. Keywords are normalized to UpperCase for high-performance comparison.
* **Parser:** A recursive-descent parser that transforms tokens into a strongly-typed Abstract Syntax Tree (AST) with embedded location data.
* **Fault-Tolerance Engine:** A resynchronization layer that protects the AST from crashing on incomplete or invalid user input during active typing.

## ⏳ Roadmap

### Phase 1: Structural Intelligence (Active Focus)
* [ ] **Scope & Symbol Tracking:** Implementing a hierarchical `Scope` system to track local variables (`DECLARE`) and parameters.
* [ ] **CTE Resolution:** Linking table references in queries to definitions provided in `WITH` clauses.
* [ ] **Reference Validation:** Ability to flag "Undefined Variable" errors based on the current AST context.

### Phase 2: Advanced T-SQL Idioms
* [ ] **The OUTPUT Clause:** Support for `INSERT/UPDATE/DELETE ... OUTPUT` patterns.
* [ ] **MERGE Statement:** Implementation of the high-complexity `MERGE` syntax.
* [ ] **Type Inference:** Basic engine to determine result types of expressions for IntelliSense tooltips.

### Phase 3: Tooling & Schema
* [ ] **Expanded DDL:** Full support for `ALTER` and `DROP` across all object types.
* [ ] **Linter Mode:** Identifying anti-patterns (e.g., `SELECT *` or missing `WHERE` clauses on deletes).
* [ ] **Formatting Engine:** A structural code formatter that uses the AST to enforce style guidelines without breaking functionality.
